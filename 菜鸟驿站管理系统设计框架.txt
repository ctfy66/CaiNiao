菜鸟驿站管理系统设计框架
一、总体架构
系统分为两个主要模块：管理员模块和用户模块，采用C语言实现，基于文件系统存储数据。
二、数据结构设计
1. 包裹信息结构体 (Package)
- 货物单号 (tracking_number)
- 用户手机号 (user_phone)
- 入库时间 (arrival_time)
- 出库时间 (departure_time)
- 包裹重量 (weight)
- 特殊需求 (special_requirements)
- 取件码 (pickup_code)
- 快递公司 (courier_company)
- 收件人信息 (recipient_info)
- 寄件人信息 (sender_info)
- 包裹状态 (status): 入库、待取、已取、拒收、退回等
- 保存条件 (storage_condition): 常温、冷藏、防潮等
- 包裹尺寸 (size): 小、中、大
- 运输距离 (distance)
- 运输方式 (transport_method)
- 是否保价 (is_insured)
- 保价金额 (insured_value)
- 费用 (cost)
2. 用户信息结构体 (User)
- 用户名 (username)
- 手机号 (phone_number)
- 密码 (password)
- 付款方式 (payment_method): 现金、微信、支付宝等
- 结算方式 (settlement_method): 一件一结、月付等
- 货物数量 (package_count)
- 会员状态 (is_member)
- 包裹列表 (packages): 指向该用户所有包裹的指针数组
3. 管理员信息结构体 (Admin)
- 用户名 (username)
- 密码 (password)
- 权限级别 (permission_level)
4. 系统信息结构体 (System)
- 运营资金 (operating_funds): 初始10万元
- 用户数量 (user_count)
- 在库包裹数量 (package_in_storage_count)
- 货架空间分配 (shelf_allocation)
三、功能模块设计
1. 认证模块
•	用户登录功能
•	管理员登录功能
•	新用户注册功能
•	密码修改功能
2. 管理员模块
•	包裹管理 
o	批量导入包裹信息（从文件）
o	手动添加包裹信息
o	修改包裹信息
o	删除包裹信息
o	查询包裹信息（按单号、手机号等）
•	用户管理 
o	添加用户
o	修改用户信息
o	删除用户
o	查询用户信息
•	库存管理 
o	库存状态查询
•	异常处理 
o	包裹损坏处理
o	包裹丢失处理
o	误领/冒领处理
o	拒收处理
3. 用户模块
•	包裹查询 
o	查询自己的包裹状态
o	历史包裹记录查询
•	寄件服务 
o	新增寄件请求
o	查询寄件状态
o	取消寄件
•	收件服务 
o	查询待收包裹
o	获取取件码
•	费用查询 
o	查询待支付费用
o	查询历史支付记录
•	个人信息管理 
o	修改个人信息
o	修改付款方式和结算方式
4. 通知模块
•	入库通知发送
•	寄件状态通知
•	费用结算通知
•	异常情况通知
5. 计费模块
•	根据重量计算费用
•	根据距离计算费用
•	根据运输方式计算费用
•	特殊保存条件附加费
•	保价服务费计算
6. 文件操作模块
•	包裹信息文件读写
•	用户信息文件读写
•	系统信息文件读写
•	日志记录

四、数据流设计
1. 包裹入库流程(管理员模块)
1.	扫描/输入包裹信息
2.	系统分配存储位置
3.	更新包裹状态为"入库"
4.	向收件人发送取件通知
5.	更新系统数据
2. 包裹出库流程(用户模块)
1.	用户提供取件码或身份验证
2.	系统验证用户身份
3.	系统定位包裹
4.	更新包裹状态为"已取"
5.	结算费用（如需）
6.	更新系统数据
3. 寄件流程(用户模块)
1.	用户提交寄件申请
2.	系统生成运单号
3.	包裹信息录入
4.	费用计算与支付
5.	包裹入库
6.	向寄件人发送确认通知
4. 异常处理流程
1.	发现异常（损坏/丢失/误领/拒收）
2.	记录异常信息
3.	启动相应处理流程
4.	进行必要赔偿
5.	更新包裹状态
6.	通知相关用户
五、安全措施设计
1.	用户身份验证（密码加密存储）
2.	取件码验证
3.	手机号验证
六、接口设计
1.	用户界面（命令行操作菜单）
七、系统流程图
系统启动,加载货物以及用户文件 -> 选择用户登录, 管理员登录,或新用户注册-> 选择使用相应的功能(如查询快递,取件,寄件等) -> 重复上一过程直至用户退出->保存数据文件.
